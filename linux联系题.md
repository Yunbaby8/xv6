# Linux 内核入门 10 个经典练习题

------

## 题 1：编译并启动一个新内核

### 目标

把当前系统从原来的内核，切到你自己编译的内核。

### 练什么

- 内核源码获取
- 配置、编译、安装
- grub 启动选择
- 回滚能力

### 大概涉及

- `.config`
- `make menuconfig`
- `make`
- `make modules_install`
- `make install`

### 怎么验证

```
uname -r
```

看到新内核版本。

### 难度

**1/10**

### 为什么必须先做

因为这是后面一切练习的基础。
 连“换内核”都没跑通，后面改源码都没有意义。

------

## 题 2：给内核版本加你自己的后缀

### 目标

让 `uname -r` 里带上你的标记，比如：

- `-yunhe`
- `-lab1`

### 练什么

- 最小改动闭环
- 内核构建产物验证
- 配置项和版本机制

### 大概涉及

通常会接触到：

- 内核版本配置
- `EXTRAVERSION` 或 local version 相关配置

### 怎么验证

```
uname -r
```

### 难度

**1/10**

### 意义

这是最稳的新手第一改。
 改完马上能看到结果，建立信心。

------

## 题 3：在启动日志里加一条自己的 printk

### 目标

系统启动后，在 `dmesg` 里看到你自己打印的一行日志。

### 练什么

- 找启动路径
- 使用 `printk`
- 看内核日志
- 验证代码真的执行到了

### 大概改哪里

可以挑一个相对早期但不太危险的初始化路径插一句日志。
 你前期不用追求“最正确的位置”，先找一个容易验证的位置。

### 怎么验证

```
dmesg | grep 你打印的关键字
```

### 难度

**2/10**

### 意义

开始建立“源码改动 → 启动执行 → 日志可见”的链路。

------

## 题 4：给 `/proc` 新增一个只读节点

### 目标

新增一个类似：

```
/proc/myinfo
```

读取时输出固定字符串，比如：

- your name
- build time
- test message

### 练什么

- procfs 基础
- 用户态读取内核导出信息
- 简单文件接口

### 大概改哪里

- `procfs` 注册逻辑
- read 回调函数

### 怎么验证

```
cat /proc/myinfo
```

### 难度

**3/10**

### 意义

这是第一个真正把“内核信息 → 用户态可读”串起来的练习。

------

## 题 5：让 `/proc` 节点输出动态信息

### 目标

不是输出固定字符串，而是输出动态内容，比如：

- jiffies
- 当前运行时间
- 当前进程数量
- 当前页大小等

### 练什么

- 内核里取系统状态
- 格式化输出
- procfs 的实用写法

### 大概改哪里

还是 `/proc` 那条线，只是把输出内容变成动态信息。

### 怎么验证

多读几次看看输出会不会变化：

```
cat /proc/myinfo
sleep 1
cat /proc/myinfo
```

### 难度

**3/10**

### 意义

让你从“静态实验”进入“观测内核状态”。

------

## 题 6：写一个 hello 内核模块

### 目标

写一个最简单模块：

- 加载时打印一条日志
- 卸载时打印一条日志

### 练什么

- 模块机制
- `insmod/rmmod`
- `lsmod/modinfo`
- 内核外部模块编译

### 大概涉及

- `module_init`
- `module_exit`
- `printk`

### 怎么验证

```
sudo insmod hello.ko
dmesg | tail
sudo rmmod hello
dmesg | tail
```

### 难度

**2/10**

### 意义

这是现实里非常常见的内核开发入门方式。
 风险低，反馈快。

------

## 题 7：给模块加参数

### 目标

加载模块时传参数，比如：

```
sudo insmod hello.ko myvalue=123 name=test
```

然后日志里打印出来。

### 练什么

- 模块参数机制
- 用户态向内核传值
- 参数类型和验证

### 大概涉及

- `module_param`

### 怎么验证

```
sudo insmod hello.ko myvalue=123
dmesg | tail
```

### 难度

**3/10**

### 意义

你会开始感受到“模块像个可配置的小内核组件”。

------

## 题 8：用 sysfs 或 procfs 做一个可读写接口

### 目标

不只是读，还要支持写。
 例如：

- 读出来是当前值
- `echo 100 > 某节点` 后，值被修改

### 练什么

- 用户态写入内核
- 简单配置接口
- 内核状态可控

### 大概改哪里

- `/proc` write 回调
   或者
- `sysfs` 属性接口

### 怎么验证

```
cat /proc/myctl
echo 100 | sudo tee /proc/myctl
cat /proc/myctl
```

### 难度

**4/10**

### 意义

这一步比只读接口更像真实系统里的控制入口。

------

## 题 9：实现一个最小字符设备驱动

### 目标

实现一个简单 `/dev/mydev`，支持：

- open
- read
- write

比如：

```
echo hello > /dev/mydev
cat /dev/mydev
```

### 练什么

- 设备号
- `file_operations`
- 用户态/内核态数据拷贝
- 设备驱动基本骨架

### 大概涉及

- `alloc_chrdev_region`
- `cdev_init`
- `cdev_add`
- `copy_to_user`
- `copy_from_user`

### 怎么验证

- 创建设备节点
- 写进去再读出来
- 看日志确认 read/write 被调用

### 难度

**6/10**

### 意义

这是正式进入 Linux 驱动模型的第一步。
 做到这里，你已经不是“只会打印日志”了。

------

## 题 10：新增一个简单系统调用

### 目标

新增一个 syscall，比如：

- 返回固定整数
- 返回当前 jiffies
- 返回一个简单计数值

用户态写个小测试程序调用它。

### 练什么

- syscall 入口链路
- 内核与用户 ABI 接口
- 架构相关修改
- 用户态测试

### 大概涉及

- syscall table
- syscall 声明
- 内核实现
- 用户态封装/测试程序

### 怎么验证

用户态程序调用后拿到正确返回值。

### 难度

**7/10**

### 意义

这是非常经典，但不适合最开始。
 做到这一步，你会明显更理解“用户态如何进内核”。

# 额外加 3 个进阶题

如果前面 10 个走顺了，可以继续这几个。

------

## 进阶题 A：增加一个定时器/周期性打印模块

### 目标

模块加载后每隔几秒打印一条日志，卸载时停止。

### 练什么

- timer
- workqueue 或 delayed work
- 生命周期管理

### 难度

**5/10**

------

## 进阶题 B：做一个内核内存观测器

### 目标

通过 `/proc` 或模块输出：

- 空闲页数量
- 内存统计
- slab 信息中的一部分

### 练什么

- 内存子系统观测
- 内核统计接口

### 难度

**5/10**

------

## 进阶题 C：改调度相关行为的一个小点

### 目标

不是重写调度器，而是做一个很小的观测或微改动。
 比如增加一条调度路径日志，或者统计某类调度事件次数。

### 练什么

- 调度路径
- 关键热点函数
- 高风险区代码的最小改动

### 难度

**8/10**

### 注意

这个不要太早碰。

# 我最建议你的练习顺序

如果你现在就准备开始，我建议按这个顺序做：

## 第一阶段：最小闭环

1. 编译并启动新内核
2. 改版本后缀
3. 加启动日志

## 第二阶段：内核对用户导出信息

1. `/proc` 只读节点
2. `/proc` 动态输出
3. 模块 hello
4. 模块参数

## 第三阶段：接口与驱动

1. 可读写 proc/sysfs
2. 字符设备驱动

## 第四阶段：用户态进入内核

1. 新增 syscall

这个顺序非常稳。

# 每个练习你都要写的记录模板

这个你一定要养成习惯。每做一个题，记录这 5 项：

## 1. 目标

我这次想验证什么？

## 2. 改动点

改了哪个文件、哪个函数、哪段逻辑？

## 3. 编译与启动

用了什么命令？

## 4. 验证方法

用什么命令确认成功？

## 5. 问题与复盘

遇到了什么错误，下次怎么避免？

这会让你的学习沉淀很快变成“可复用经验”。
