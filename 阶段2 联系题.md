

| 模块                                  | 建议时间  | 学习目标                                           | 学习方式                                              | 推荐实验 / 项目                                              | 达标标准                                                     |
| ------------------------------------- | --------- | -------------------------------------------------- | ----------------------------------------------------- | ------------------------------------------------------------ | ------------------------------------------------------------ |
| 1. 系统调用与用户/内核边界            | 2 周      | 理解用户态如何主动进入内核，参数和返回值怎么传递   | 先做实验，再画调用链，再看少量源码                    | `my_add(int a, int b)`；`my_task_info()`；`my_get_comm(char __user *buf, size_t len)` | 能解释 `syscall number → syscall_64.tbl → SYSCALL_DEFINE → 返回用户态`；会用 `copy_to_user/copy_from_user` |
| 2. 进程管理与 `task_struct`           | 2 到 3 周 | 理解 Linux 如何描述进程                            | 用 procfs 导出进程信息，边做边查 `task_struct` 字段   | `/proc/my_tasks`；`/proc/my_task_state`                      | 能说清 `current`、`task_struct`、`pid`、`comm`、`state` 是什么 |
| 3. 调度器入门                         | 3 到 4 周 | 理解进程为什么会切换，Linux 大概如何选择下一个进程 | 用户态现象观察 + 内核 printk 插桩                     | `context_switch()` 限流打印；`nice` 对比实验；CPU busy loop 观察 | 能解释 `schedule()`、`prev/next`、`context switch`、`nice`、`vruntime` 的大概作用 |
| 4. 用户虚拟内存与 `mm_struct`         | 3 到 4 周 | 理解进程虚拟地址空间                               | 先观察 `/proc/<pid>/maps`，再从内核导出 `current->mm` | 用户态打印 text/data/heap/stack 地址；`/proc/my_mm`          | 能解释代码段、数据段、堆、栈、`mm_struct`、`vm_area_struct`  |
| 5. 内核内存分配                       | 2 到 3 周 | 理解内核自己如何申请内存                           | 写模块对比实验，不急着深挖页表                        | `kmalloc/vmalloc` 对比模块；打印地址并释放                   | 能说清 `kmalloc` 和 `vmalloc` 的区别，知道 `kfree/vfree`     |
| 6. 字符设备与 VFS 接口                | 3 到 4 周 | 理解用户态 `open/read/write` 怎么进入驱动          | 在你已有 `/dev/mydev` 基础上升级                      | 增加 `release`、`llseek`、自动创建设备节点                   | 能解释 `file_operations`、`struct file`、主设备号/次设备号、`/dev` 节点 |
| 7. `ioctl` 与用户/内核数据交换        | 2 到 3 周 | 理解真实驱动常用控制接口                           | 写 GET/SET 命令，练用户指针拷贝                       | `/dev/mydev` 支持 `MYDEV_SET_VALUE`、`MYDEV_GET_VALUE`       | 能正确使用 `_IOW/_IOR`、`copy_from_user/copy_to_user`，知道为什么不能直接访问用户指针 |
| 8. 定时器、中断下半部与 workqueue     | 3 到 4 周 | 理解内核异步执行机制                               | 从 timer 开始，再做 workqueue                         | timer 每秒打印；timer 触发 workqueue 执行任务                | 能解释 `jiffies`、`HZ`、timer、workqueue、为什么有些上下文不能睡眠 |
| 9. sysfs 与驱动模型入门               | 3 到 4 周 | 理解 Linux 设备模型的基本结构                      | 从字符设备自动节点过渡到 class/device/sysfs           | `/sys/class/mydev/mydev0/value` 支持读写；platform driver 入门 | 能解释 `class_create`、`device_create`、sysfs 属性、platform driver 的 `probe/remove` |
| 10. QEMU 完整 rootfs / 给虚拟机换内核 | 2 到 4 周 | 从 initramfs 过渡到更像真实系统的虚拟机环境        | 做 ext4 rootfs 镜像，用自编译 `bzImage` 启动          | QEMU 使用 `-drive` 挂载 rootfs；内核参数 `root=/dev/vda`     | 能用自己编译的内核启动一个完整 rootfs，知道内核、rootfs、启动参数之间的关系 |
| 11. 内核调试与定位问题                | 持续学习  | 提高排错能力                                       | 每个模块都加调试手段                                  | `printk`、`dmesg`、`ftrace`、`strace`、`gdb`/QEMU 调试       | 出问题时能定位是用户态、rootfs、内核配置、驱动还是 syscall 表问题 |
