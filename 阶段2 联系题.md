实验 A：syscall 参数与 current 观察实验

实现两个 syscall：

my_add(int a, int b)
my_task_info(void)

用户态测试：

syscall(549, 10, 20);
syscall(550);

内核打印：

[my_add] current=sh pid=23 a=10 b=20 result=30
[my_task_info] comm=test_task pid=31

这样你就能把：

参数传递 + 当前进程上下文

连起来。

项目 B：进程与调度观察器

你可以做一个小项目，名字叫：

process_sched_observer

功能包括：

/proc/my_tasks
/proc/my_task_state
调度切换 printk 或记录最近 N 次切换
nice 实验验证调度权重

最后你要能观察到：

pid=1 comm=init state=S
pid=23 comm=sh state=S
pid=45 comm=busy state=R

以及：

switch: sh[23] -> busy[45]
switch: busy[45] -> idle[0]
模块二做完你应该能回答
1. Linux 用什么结构描述一个进程？
2. task_struct 里 pid、comm、state 分别表示什么？
3. for_each_process() 为什么能遍历所有进程？
4. 一个进程有哪些基本状态？
5. schedule() 大概什么时候发生？
6. context_switch() 里的 prev 和 next 是什么意思？
7. nice 值为什么会影响 CPU 时间？
8. CFS 为什么不是简单平均分配 CPU？

项目 C：内存观察器

可以做成：

memory_observer

包括：

用户态程序打印 text/data/heap/stack 地址
/proc/my_mm 输出 current->mm 信息
内核模块测试 kmalloc/vmalloc 地址差异

最后你要能对比：

用户态看到的地址
/proc/<pid>/maps 里的地址
current->mm 里的地址
内核 kmalloc/vmalloc 的地址


项目 D：增强版字符设备驱动

你现在已经有 /dev/mydev。

下一步不要只单独做 ioctl，而是把它升级成：

/dev/mydev
支持 open
支持 read
支持 write
支持 ioctl get/set
支持自动创建设备节点
后续支持 poll/mmap

第一阶段先做：

ioctl GET_VALUE
ioctl SET_VALUE

用户态：

int val = 123;
ioctl(fd, MYDEV_SET_VALUE, &val);

val = 0;
ioctl(fd, MYDEV_GET_VALUE, &val);
printf("val=%d\n", val);

内核态：

copy_from_user(&kernel_value, argp, sizeof(int));
copy_to_user(argp, &kernel_value, sizeof(int));
模块四做完你应该能回答
1. read/write/ioctl 有什么区别？
2. 为什么不能直接访问用户态指针？
3. copy_from_user 是从哪里 copy 到哪里？
4. copy_to_user 是从哪里 copy 到哪里？
5. ioctl 为什么常用于设备控制？
6. 字符设备驱动的 file_operations 是怎么被 VFS 调用的？
