```
模块加载：
module_init() -> init 函数

模块卸载：
module_exit() -> exit 函数

创建 proc：
proc_create() -> proc_ops -> open -> single_open() -> show()

读取 proc：
cat -> open -> read -> show -> seq_printf()

删除 proc：
remove_proc_entry()
```

## 3. 具体代码

```c
#include <linux/init.h>
#include <linux/module.h>
#include <linux/kernel.h>
#include <linux/proc_fs.h>
#include <linux/seq_file.h>
#include <linux/jiffies.h>
#include <linux/utsname.h>
#include <linux/errno.h>

#define PROC_NAME "hello_module"

static int myvalue = 0;
static char *name = "default";

module_param(myvalue, int, 0644);
MODULE_PARM_DESC(myvalue, "An integer value passed to hello_module");

module_param(name, charp, 0644);
MODULE_PARM_DESC(name, "A string name passed to hello_module");

static int hello_module_show(struct seq_file *m, void *v)
{
    seq_printf(m, "Hello from hello_module\n");
    seq_printf(m, "myvalue: %d\n", myvalue);
    seq_printf(m, "name: %s\n", name);
    seq_printf(m, "jiffies: %lu\n", jiffies);
    seq_printf(m, "release: %s\n", init_uts_ns.name.release);

    return 0;
}

static int hello_module_open(struct inode *inode, struct file *file)
{
    return single_open(file, hello_module_show, NULL);
}

static const struct proc_ops hello_module_proc_ops = {
    .proc_open    = hello_module_open,
    .proc_read    = seq_read,
    .proc_lseek   = seq_lseek,
    .proc_release = single_release,
};

static int __init hello_module_init(void)
{
    if (myvalue < 0) {
        pr_err("hello_module: invalid myvalue=%d, must be >= 0\n", myvalue);
        return -EINVAL;
    }

    if (!proc_create(PROC_NAME, 0444, NULL, &hello_module_proc_ops)) {
        pr_err("hello_module: failed to create /proc/%s\n", PROC_NAME);
        return -ENOMEM;
    }

    pr_info("hello_module: loaded\n");
    pr_info("hello_module: myvalue=%d\n", myvalue);
    pr_info("hello_module: name=%s\n", name);
    pr_info("hello_module: /proc/%s created\n", PROC_NAME);

    return 0;
}

static void __exit hello_module_exit(void)
{
    remove_proc_entry(PROC_NAME, NULL);
    pr_info("hello_module: /proc/%s removed\n", PROC_NAME);
    pr_info("hello_module: unloaded\n");
}

module_init(hello_module_init);
module_exit(hello_module_exit);

MODULE_LICENSE("GPL");
MODULE_AUTHOR("Eric Zhou");
MODULE_DESCRIPTION("A hello module with procfs entry and parameters");
```

------

## 4. 代码分析

### 4.1 新增模块参数变量

```c
static int myvalue = 0;
static char *name = "default";
```

这两个变量是模块内部状态。

默认情况下：

```text
myvalue = 0
name = default
```

如果加载模块时不传参数，就使用默认值。

------

### 4.2 `module_param()`

```c
module_param(myvalue, int, 0644);
```

作用：
把 `myvalue` 这个 C 变量注册成模块参数。

含义：

| 项目      | 含义                 |
| --------- | -------------------- |
| `myvalue` | 参数名，也对应变量名 |
| `int`     | 参数类型             |
| `0644`    | sysfs 中参数文件权限 |

加载时可以传：

```sh
insmod hello_module.ko myvalue=123
```

然后模块内部的：

```c
myvalue
```

就会变成：

```c
123
```

------

### 4.3 字符串参数

```c
module_param(name, charp, 0644);
```

作用：
把 `name` 注册成字符串参数。

`charp` 表示字符串指针参数。

加载时可以传：

```sh
insmod hello_module.ko name=yunhe
```

模块内部的：

```c
name
```

就会变成：

```text
yunhe
```

------

### 4.4 `MODULE_PARM_DESC()`

```c
MODULE_PARM_DESC(myvalue, "An integer value passed to hello_module");
MODULE_PARM_DESC(name, "A string name passed to hello_module");
```

作用：
给模块参数添加描述信息。

这不是模块运行必需的逻辑，但可以通过：

```sh
modinfo hello_module.ko
```

查看参数说明。

------

### 4.5 参数校验

```c
if (myvalue < 0) {
    pr_err("hello_module: invalid myvalue=%d, must be >= 0\n", myvalue);
    return -EINVAL;
}
```

作用：
检查用户传入的参数是否合法。

如果用户传：

```sh
insmod hello_module.ko myvalue=-1
```

模块会认为参数无效，并返回：

```c
-EINVAL
```

`-EINVAL` 表示：

```text
Invalid argument，参数无效
```

返回错误后，模块加载失败，后面的 `proc_create()` 不会执行，`/proc/hello_module` 也不会出现。

------

### 4.6 proc 输出新增参数显示

```c
seq_printf(m, "myvalue: %d\n", myvalue);
seq_printf(m, "name: %s\n", name);
```

作用：
用户执行：

```sh
cat /proc/hello_module
```

时，可以看到当前模块参数值。

这让 `/proc/hello_module` 不只是固定输出，而是能显示模块加载时的配置状态。

------

### 4.7 模块加载函数

```c
static int __init hello_module_init(void)
```

作用：
模块加载时执行。

这个函数现在做三件事：

1. 检查 `myvalue` 是否合法
2. 创建 `/proc/hello_module`
3. 打印当前参数到内核日志

------

### 4.8 模块卸载函数

```c
static void __exit hello_module_exit(void)
```

作用：
模块卸载时执行。

它负责删除：

```sh
/proc/hello_module
```

避免模块卸载后留下无效 proc 节点。

------

## 5. 内核操作流程

### 5.1 用户传参加载模块时

用户执行：

```sh
insmod /root/hello_module.ko myvalue=123 name=yunhe
```

内核大致做这些事：

```text
insmod 读取 hello_module.ko
  ↓
请求内核加载模块
  ↓
内核解析模块参数
  ↓
找到 module_param 注册的 myvalue 和 name
  ↓
把 myvalue 设置成 123
  ↓
把 name 设置成 "yunhe"
  ↓
调用 hello_module_init()
```

------

### 5.2 进入 `hello_module_init()`

```text
hello_module_init()
  ↓
检查 myvalue 是否小于 0
  ↓
如果合法，继续
  ↓
proc_create(PROC_NAME, ...)
  ↓
创建 /proc/hello_module
  ↓
打印 myvalue 和 name
  ↓
模块加载成功
```

如果参数非法：

```text
myvalue < 0
  ↓
打印错误日志
  ↓
return -EINVAL
  ↓
模块加载失败
  ↓
/proc/hello_module 不会创建
```

------

### 5.3 用户读取 `/proc/hello_module`

用户执行：

```sh
cat /proc/hello_module
```

内核流程：

```text
cat open("/proc/hello_module")
  ↓
procfs 找到 hello_module 节点
  ↓
调用 hello_module_open()
  ↓
single_open() 绑定 hello_module_show()
  ↓
cat read()
  ↓
seq_read()
  ↓
调用 hello_module_show()
  ↓
seq_printf() 输出 myvalue、name、jiffies、release
  ↓
内容返回给用户态
```

------

### 5.4 用户卸载模块

用户执行：

```sh
rmmod hello_module
```

内核流程：

```text
rmmod 请求卸载模块
  ↓
内核检查模块是否仍在使用
  ↓
如果引用计数为 0，允许卸载
  ↓
调用 hello_module_exit()
  ↓
remove_proc_entry(PROC_NAME, NULL)
  ↓
删除 /proc/hello_module
  ↓
释放模块资源
  ↓
模块卸载完成
```

------

## 6. 实验意义

这个实验让模块从“固定功能”变成了“可配置功能”。

之前模块加载时行为固定：

```sh
insmod hello_module.ko
```

现在可以在加载时传入配置：

```sh
insmod hello_module.ko myvalue=123 name=yunhe
```

这非常接近真实驱动的工作方式。

例如真实驱动中可能有：

```text
debug=1
bus_id=3
timeout_ms=500
mode=auto
```

模块参数常用于：

- 控制调试开关
- 设置设备编号
- 设置超时时间
- 选择运行模式
- 调整初始化行为

加载时：

## 8. 一句话总结

本实验在已有模块动态创建 `/proc/hello_module` 的基础上增加了模块参数机制，通过 `module_param()` 使用户可以在 `insmod` 加载模块时传入 `myvalue` 和 `name`，并在模块初始化日志和 `/proc/hello_module` 中验证参数值，从而理解了用户态向内核模块传递配置参数的基本方法。
