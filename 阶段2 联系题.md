# 下一阶段 10 个 Linux 内核实验

## 1. 带参数系统调用实验

**目标：** 新增一个 syscall：

```c
my_add(int a, int b)
```

用户态调用：

```c
syscall(549, 10, 20);
```

返回：

```text
30
```

**练什么：**

```text
SYSCALL_DEFINE2
系统调用参数传递
用户态参数如何进入内核
syscall ABI
```

**为什么先做：**
你刚做完无参数 syscall，现在做带参数 syscall 是最自然的升级。

------

## 2. 当前进程信息 syscall 实验

**目标：** 新增一个 syscall，返回当前进程 pid，并在内核日志打印当前进程名：

```c
current->pid
current->comm
```

用户态运行后：

```text
pid = xxx
```

`dmesg` 看到：

```text
comm=test_my_task pid=xxx
```

**练什么：**

```text
current 宏
task_struct
pid
comm
当前进程上下文
```

**意义：**
这是你进入进程管理和调度器的入口。

------

## 3. `/proc/my_tasks` 进程列表实验

**目标：** 新增一个 proc 节点：

```text
/proc/my_tasks
```

执行：

```bash
cat /proc/my_tasks
```

输出类似：

```text
pid=1 comm=init
pid=23 comm=sh
pid=45 comm=test
```

**练什么：**

```text
for_each_process()
task_struct 链表
procfs
seq_file
进程遍历
```

**意义：**
你会第一次从内核视角看到“系统里有哪些进程”。

------

## 4. `/proc/my_task_state` 进程状态实验

**目标：** 在 procfs 中输出每个进程的状态，比如：

```text
pid=1 comm=init state=S
pid=35 comm=sh state=R
```

**练什么：**

```text
task->__state
TASK_RUNNING
TASK_INTERRUPTIBLE
TASK_UNINTERRUPTIBLE
进程状态
```

**意义：**
这是理解调度器之前必须懂的基础：进程不是一直在运行，它会有不同状态。

------

## 5. 简单调度切换 printk 插桩实验

**目标：** 在调度路径中打印少量进程切换日志：

```text
switch: sh[23] -> busy[45]
switch: busy[45] -> idle[0]
```

建议位置：

```text
kernel/sched/core.c
```

重点函数：

```c
__schedule()
context_switch()
```

**练什么：**

```text
prev
next
context switch
schedule()
调度切换路径
```

**注意：**
一定要限制打印次数，比如只打印前 20 次，否则日志会爆炸。

------

## 6. nice 值影响调度实验

**目标：** 写两个用户态 CPU 忙循环程序，一个普通运行，一个用 nice 低优先级运行：

```bash
./busy &
nice -n 10 ./busy &
```

观察 CPU 时间差异。

可以看：

```bash
top
ps -o pid,ni,comm
cat /proc/<pid>/stat
```

**练什么：**

```text
nice
优先级
CFS
vruntime
CPU 时间片/权重
```

**意义：**
让你从现象上理解 Linux 调度器不是“平均分配”，而是按权重调度。

------

## 7. 用户态虚拟地址观察实验

**目标：** 写一个用户态程序打印：

```text
代码段地址
全局变量地址
堆地址
栈地址
```

然后查看：

```bash

```

**练什么：**

```text
虚拟地址空间
代码段
数据段
堆
栈
/proc/<pid>/maps
```

**意义：**
这是进入内存管理最好的入口。

## 8. `/proc/my_mm` 当前进程内存信息实验

**目标：** 新增 proc 节点：

```text
/proc/my_mm
```

输出当前进程的内存布局信息：

```text
start_code
end_code
start_data
end_data
start_brk
brk
start_stack
```

核心结构：

```c
current->mm
```

**练什么：**

```text
mm_struct
进程虚拟地址空间
用户进程内存布局
```

**意义：**
你会从源码层面认识 Linux 是怎么描述一个进程的内存空间的。

------

## 9. kmalloc / vmalloc 对比实验

**目标：** 写一个内核模块，分别申请：

```c
kmalloc()
vmalloc()
```

打印两个地址，并在卸载模块时释放：

```c
kfree()
vfree()
```

**练什么：**

```text
内核内存分配
kmalloc
vmalloc
物理连续和虚拟连续
模块资源释放
```

**意义：**
这是内核内存管理的入门实验。

------

## 10. 字符设备 ioctl 实验

**目标：** 升级你之前的 `/dev/mydev`，增加：

```c
.unlocked_ioctl = mydev_ioctl
```

支持用户态：

```c
ioctl(fd, MYDEV_SET_VALUE, &val);
ioctl(fd, MYDEV_GET_VALUE, &val);
```

**练什么：**

```text
ioctl
copy_from_user
copy_to_user
用户指针
驱动控制接口
```

**意义：**
这是字符设备驱动从“玩具版”走向“真实驱动”的关键一步。

# 建议你按这个顺序做

```text
1. 带参数 syscall
2. 当前进程信息 syscall
3. /proc/my_tasks
4. /proc/my_task_state
5. 调度切换 printk
6. nice 调度实验
7. 用户态虚拟地址观察
8. /proc/my_mm
9. kmalloc / vmalloc
10. 字符设备 ioctl
```

这 10 个做完，你会建立三条主线：

```text
系统调用 → 进程 → 调度
procfs → task_struct/mm_struct → 内核数据结构
字符设备 → ioctl → 用户态/内核态数据交换
```

下一阶段的核心目标不是“多写几个小功能”，而是开始真正理解：

```text
Linux 如何描述进程
Linux 如何调度进程
Linux 如何管理进程内存
Linux 如何让用户程序和内核安全交换数据
```
