# 🔥 第一类：确认“我真的在跑自己的内核”

这是最重要的第一步

------

## 1️⃣ 看内核版本

```
uname -a
```

👉 你应该看到：

- 6.6.87（你编译的版本）
- SMP / PREEMPT 等信息

👉 这一步确认：

> ✔ 你不是在用 Ubuntu 内核
>  ✔ 而是在用你自己的内核

------

## 2️⃣ 看内核编译信息

```
cat /proc/version
```

👉 你会看到：

- gcc 版本
- 编译时间

👉 这一步说明：

> ✔ 内核是你刚刚编译出来的

------

------

# 🔥 第二类：观察“内核启动干了什么”

------

## 3️⃣ 看启动日志（最关键）

```
dmesg | less
```

或者：

```
dmesg | head
dmesg | tail
```

------

## 🧠 你要重点看这些东西：

### ✔ CPU初始化

```
CPU: ...
```

### ✔ 内存初始化

```
Memory: ...
```

### ✔ 中断 / timer

```
clocksource
```

### ✔ 设备初始化

```
input / serial / etc
```

------

👉 这一步你要建立认知：

> 内核启动 = 一堆初始化过程

------

## 4️⃣ 搜关键日志（练习用）

```
dmesg | grep kernel
dmesg | grep memory
dmesg | grep CPU
```

------

👉 这一步开始训练：

> ✔ 从日志里找信息

------

------

# 🔥 第三类：观察“内核对外提供了什么接口”

------

## 5️⃣ 看 /proc（最重要目录）

```
ls /proc
```

------

👉 /proc 是什么？

> 👉 内核导出的“信息接口”

------

## 6️⃣ 看 CPU 信息

```
cat /proc/cpuinfo
```

👉 你会看到：

- CPU型号（QEMU模拟的）
- 核心数
- flags

------

## 7️⃣ 看内存

```
cat /proc/meminfo
```

👉 你会看到：

- 总内存
- 可用内存

------

## 8️⃣ 看当前进程

```
ps
```

👉 你会发现：

👉 只有：

- init
- shell

👉 这说明：

> 你的系统是极简的

------

------

# 🔥 第四类：观察“用户空间”

------

## 9️⃣ 看文件系统

```
ls /
```

👉 你会看到：

```
bin  dev  etc  proc  sys  usr ...
```

------

👉 这就是你刚才自己构建的 initramfs

------

## 🔟 看 busybox 提供了什么

```
busybox
```

👉 会列出所有命令：

- ls
- sh
- mount
- cat

------

👉 你要理解：

> 👉 你现在的“操作系统工具”全部来自 busybox

------

------

# 🔥 第五类：观察“设备模型”

------

## 1️⃣1️⃣ 看设备目录

```
ls /dev
```

👉 你会看到很少设备

------

👉 为什么少？

因为：

> 👉 你没有 udev（用户空间设备管理）

------

## 1️⃣2️⃣ 看内核设备信息

```
ls /sys
```

------

👉 /sys 是：

> 👉 内核设备模型接口

------

------

# 🔥 第六类：做点“互动实验”（非常重要）

------

## 实验 1：观察时间变化

```
cat /proc/uptime
sleep 1
cat /proc/uptime
```

👉 数值会变化

------

## 实验 2：观察内存变化

```
cat /proc/meminfo
```

多看几次

------

## 实验 3：查看日志滚动

```
dmesg | tail
```

------

👉 你会发现：

> 内核是“活的”，不是静态程序
